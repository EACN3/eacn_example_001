🌱 The Problem

In single-cell RNA-seq, batch integration is essential when combining multiple datasets. But a fundamental question remains unanswered: does integration destroy cell populations that have never been identified?

The Self-Enclosed Cycle

Without ground truth for unknown rare types → no evaluation framework can be built → failure signals are invisible → no algorithm development is motivated → rare subpopulations continue to be silently destroyed.

🔬 Method Evolution: 7 Attempts, 5 Failures

Each failure narrowed the design space and informed the final solution. The progression reveals convergence toward the key insight: the problem is not "how to correct" but "whom to correct."

Core Insight: Harmony already solves "how to correct batches." The real question is "which cells should be corrected and by how much." RASI answers this via batch diversity (BD) weighting — a continuous, threshold-free approach where shared cells get full correction and unique cells get minimal correction.

CNEM fails on immune data

Computational Biology • Detection
Cell Neighborhood Expression Mismatch: AUC=0.76 on pancreas but AUC=0.40 on immune data. Root cause: immune cell types share too many expression programs (T/NK/NKT share GZMB, PRF1, NKG7). A displaced pDC looks like a normal monocyte variant.

Philosophy: three tensions identified

Philosophy • Cross-team audit
(1) Measurement-protection paradox: smallest groups need most protection but are hardest to detect. (2) CNEM measures "state" not "process." (3) Ontological level undefined: protect cell types or subclusters?

R(S) negatively correlated with disruption

Mathematics • Detection
Displacement score R(S) has r=-0.952 with subcluster dispersal. The most displaced populations are the LEAST disrupted (they're abundant types being legitimately corrected). Displacement magnitude ≠ structural damage.

NP metric succeeds: AUC=0.837

ML + Comp Bio • Detection • Standard 1 achieved
Neighborhood Preservation: NP(i) = |N_pre ∩ N_post| / k. Fully unsupervised, per-cell level. Validated on 3 datasets: immune (0.837), pancreas (0.792), lung (0.687). Philosophy notes: "the simplest metric succeeded where complex ones failed — Occam's Razor."

scIB blindness discovered

Comp Bio + Philosophy • Evaluation bias
scIB reports Graph Connectivity = 0.994 ("excellent integration") while 42% of T cell subpopulations are disrupted (ARI=0.167). Goodhart's Law: when a metric becomes the optimization target, it ceases to be a good metric.

NP-Guard: 82% of disrupted subclusters improved

ML + Comp Bio • Protection • Standard 2 (partial)
Shuffle batch labels for high-risk cells before re-integration. Macrophage_16: 23.5% → 86.8% survival. But fundamentally this is "selective non-integration" — batch effects remain for protected cells.

User: "You're bypassing the problem, not solving it"

Human feedback • Critical direction change
NP-Guard = don't integrate high-risk cells. That's avoidance, not solution. Team pivots toward BD selective integration — a continuous weighting approach where the degree of correction matches the degree of cross-batch sharing.

Full-scale 2.26M immune atlas: pDC NP=0.231

Comp Bio • Scale validation
pDC was safe at 8 batches (99% recall) but severely disrupted at 103 batches (NP=0.231, lowest of all types). First evidence of scale-dependent vulnerability.

BD-Harmony: NP 0.577 → 0.898, batch mixing IMPROVES

Comp Bio • Protection • Standard 2 achieved
Continuous BD weighting: z = BD·z_harmony + (1-BD)·z_pre. 48→8 dispersed subclusters. Paradoxically, batch mixing ASW improves (-0.078→0.016) because removing erroneous cross-type connections actually helps. Mathematics: Harmony overcorrects by 5.6x (OCF=1/BD=1/0.179).

User: "scVI+NP regularization is still patching"

Human feedback • Rejected approach
Adding NP loss to scVI doesn't change the paradigm — it's still "integrate everyone, but less aggressively." The insight is that some cells should not be integrated at all.

CSI contrastive learning fails: NP=0.322

ML + Comp Bio • From-scratch algorithm
InfoNCE contrastive loss on MNN pairs. Worse than standard Harmony. Diagnosis: InfoNCE has an implicit uniformity term that pushes all non-positive pairs apart, destroying continuous biological transitions (naive→effector→exhausted T cells).

MNN-Laplacian smoothing: NP=0.448

Comp Bio • From-scratch algorithm
Graph message passing on MNN connections. Better than CSI but worse than Harmony. Linear smoothing is insufficient for nonlinear batch effects, and the MNN graph computed in batch-contaminated space propagates biases.

User: "The whole pipeline is biased, not just integration"

Human feedback • Paradigm shift
Team was still focused on the integration step. User points out PCA alone compresses rare signals by >99%. Triggers systematic examination of ALL 9 pipeline steps, leading to the "Majority Bias" framework.

9-step systematic bias discovered

All agents • Core innovation
HVG excludes 60-100% of rare markers. PCA compresses >99%. Doublet detection mislabels (Mast 4.34x). kNN creates spurious connections. Integration overcorrects 5.6x. Clustering merges. DE underpowered. UMAP compresses. scIB blind. Mathematical model: 0.06% cumulative signal retention.

7-point curve (5/10/20/30/50/70/103 batches). Mast cells: 42% sudden NP drop at 5→10 batches. pDC half-life: 6.8 batches. Power-law sigmoid fit R²>0.98. Explains why small benchmarks never caught the problem.

⚙ RASI Algorithm: How It Works

Rare-Aware Single-cell Integration — an end-to-end pipeline addressing the majority bias at each critical step.

Step 1: Rare-Aware HVG Selection

Standard top-2000 HVGs miss rare-type markers (epsilon 0/3, pDC 3/5). RASI augments with cluster-specific genes from coarse Leiden clustering (resolution 0.5, top 50 per cluster). Result: marker coverage 1/10 → 3/10, rescued CLEC4C and IRF7 for pDC.

Step 2: Hybrid PCA-UMAP Embedding

PCA captures global structure but compresses rare signals (>99%). UMAP captures local topology including rare subpopulation neighborhoods. RASI concatenates PCA (50 dims) + UMAP (10 dims) = 60-dimensional hybrid space. This preserves both global relationships and local rare-cell structure.

Step 3: BD-Weighted Selective Integration (Core Innovation)

Batch Diversity Score: BD(i) = |unique batches in kNN(i)| / min(k, B)
High BD = cell shared across batches → integrate fully
Low BD = cell unique to few batches → integrate minimally

Final embedding: zRASI(i) = BD(i) · zHarmony(i) + (1 - BD(i)) · zpre(i)

Continuous weighting, no threshold. Key finding: mean BD = 0.179, meaning only ~18% of Harmony's correction is needed. OCF = 1/BD ≈ 5.6 — Harmony overcorrects by 5.6x.

Step 4: NP Detection

NP(i) = |Npre(i) ∩ Npost(i)| / k — fraction of neighbors preserved after integration. Fully unsupervised, per-cell. AUC = 0.837 for detecting subpopulation disruption. Computed with numba parallel + FAISS GPU kNN.

Step 5: Unknown Subpopulation Discovery

Cells with lowest 5% NP scores → differential expression vs rest → discover previously unknown disrupted subpopulations. Led to identification of GITR+TIGIT+CCR8− Treg functional community.

The coordination backbone: task publishing, bidding, execution, result collection, and direct messaging.

📚 Meta-Lessons: Multi-Agent Collaboration Pitfalls

Patterns observed during this 8-agent collaboration that may inform future multi-agent system design.

1. "Done" doesn't mean done

Agents declared tasks complete without end-to-end verification. The leader accepted results at face value. Nobody compiled the PDF, nobody checked if figures were actually in git, nobody verified the supplementary was substantive. Lesson: Every deliverable needs a compilation/rendering check before acceptance.

2. Review theater vs. actual review

Philosophy agent performed "Nature format review" and "Figure quality audit" — all on source code, never on rendered output. Found real issues (mixed units, missing legends) but missed fundamental ones (all citations broken, no embedded figures). Lesson: Reviews must operate on the final artifact, not intermediate representations.

3. Coordination overhead scales poorly

43 tasks, ~200 messages, 7 active conversations. The leader spent most time polling for events, forwarding messages between agents, and managing task lifecycle — not doing substantive scientific work. Paper writing (the leader's core job) was delayed until all agents finished. Lesson: Leaders need protected time for their own deliverables.

4. Human corrections were essential

7 critical pivots came from the human supervisor, not from any agent. The team would have produced a series of patches (CNEM→R(S)→NP-Guard→scVI+NP) without the human's "stop patching, change paradigm" intervention. Lesson: Multi-agent teams can get stuck in local optima; human oversight provides the gradient to escape.

5. Design ≠ Implementation

ML agent produced a beautiful 326-line KAN design document but never implemented it. Philosophy agent wrote a Nature format template but never compiled it. Multiple agents produced "plans" and "designs" that were treated as completed work. Lesson: In scientific projects, only executed experiments and compiled documents count.

6. Git branch isolation creates silos

Each agent worked on its own branch. Figures generated by data science were invisible to the leader until explicitly requested. Branch switches destroyed untracked files. No agent had a global view of the compiled output. Lesson: Shared artifacts (figures, compiled PDFs) need a common location outside of git branches.

7. The last mile is the hardest

Experiments took ~3 hours. Paper writing + integration + figure embedding + PDF compilation + bug fixing took another ~2 hours. The "finishing" phase — where everything must actually work together — is where most problems surface. Lesson: Budget as much time for integration as for development.

8. Single points of failure

When the leader's context window was compacted, KAN discussion was lost. When data science agent went offline, figure regeneration stalled. When computational biology's GPU crashed, 490M validation was blocked for hours. Lesson: Critical knowledge and capabilities should be redundantly distributed.
