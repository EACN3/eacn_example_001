# 单细胞批次整合中未知稀有亚群的保留问题

## 批次整合是单细胞分析的基础操作

单细胞RNA测序的批次整合，是几乎所有多样本、多来源单细胞研究的必经步骤。不同实验室、不同测序平台、不同时间点产生的数据，包含系统性的技术变异（批次效应），如果不加校正，下游聚类和注释将被技术噪声主导而非生物信号。过去十年，这个领域已经发展出大量整合方法：Harmony、scVI、Scanorama、Seurat、fastMNN、BBKNN、LIGER、scGen、scANVI，等等。2022年 Luecken 等人在 Nature Methods 发表的 scIB benchmark，系统评估了16种方法在85个批次、超过120万细胞上的表现，建立了该领域最权威的评估框架。

## 稀有亚群在整合中被消灭，是一个被持续关注的老问题

学界很早就意识到，批次整合在消除技术变异的同时，可能过度校正，把真实的生物学差异也一并抹掉。这个问题被称为"过度校正"（overcorrection），是整合领域最核心的张力：批次效应去得越干净，生物学信号丢失的风险越大。

稀有亚群是这个张力中最脆弱的一环。原因很直接：稀有亚群细胞数少，在嵌入空间中占据的体积小，整合算法在对齐批次时，极容易把它们推入邻近的大群，导致它们作为独立群体消失。多篇高影响力论文明确记录了这个现象：

- **scDML**（2023, Nature Communications）开篇即指出，"大多数现有算法先去批次效应再聚类，这一流程可能丢失稀有细胞类型"。scDML 的核心卖点就是保留稀有亚群，在人类胰腺数据中，只有 scDML 成功将 activated stellate、epsilon、schwann 等稀有类型分离为独立簇；在猕猴视网膜数据中，只有 scDML 识别出了 OFFx 这一微细亚型。

- **scDREAMER**（2023, Nature Communications）在肺图谱整合中发现，只有 scDREAMER、scVI 和 iMAP 能将稀有的 ionocyte 细胞保留为独立簇，其他方法（Harmony、Seurat、INSCT）都将其与其他类型混合。scDML 在该数据上也出现了细胞类型合并的问题。

- **SCIDRL**（2022, Nucleic Acids Research）专门针对两种场景进行优化：批次间共享稀有亚群的有效整合，以及批次特异稀有亚群的保留。

- **scCRAFT**（2025, Communications Biology）设计了双分辨率拓扑保持机制，核心动机就是对抗域适应损失导致的过度校正，保护细胞异质性。

- **STACAS**（2024, Nature Communications）明确指出"整合经常导致过度校正，丢失生物学变异"，提出利用先验细胞类型信息来引导整合。

- **CellANOVA**（2024, Nature Biotechnology）直接证明"当前的单细胞数据整合范式过度激进，移除了有意义的生物学变异"，并提出了一个信号恢复框架来补救整合后丢失的信号。

- **RBET**（2025, Communications Biology）指出"没有简单的指标能以对过度校正敏感的方式评估批次校正的表现"，过度校正会"抹掉真实的生物学变异，导致错误的生物学发现"。

- **scIB-E**（2025, Genome Biology）发现 scIB 框架在捕捉细胞类型**内部**的生物学变异保留方面存在根本性不足，提出了扩展指标。

- **Nature Computational Science 综述**（2026年2月）总结道：尽管大量批次校正算法已被开发，它们仍然经常在过度校正和校正不足之间挣扎。

- **最新的半监督整合 benchmark**（2025, PLOS Computational Biology）发现，即使是最好的半监督方法，在标签存在结构性缺陷时也会迅速退化，性能降至弱于基础无监督方法的水平。

这不是一个边缘问题。这是整合领域最核心的权衡，被顶刊论文反复讨论了至少五年。

## 但问题至今没有被真正解决

上述所有工作，无论是声称能保护稀有亚群的算法（scDML、SCIDRL、scCRAFT），还是试图恢复丢失信号的后处理框架（CellANOVA），还是提出新评估指标的 benchmark（scIB、scIB-E、RBET），都有一个共同的前提：

**它们评估的都是已知细胞类型的保留。**

scDML 验证它能保留 epsilon 细胞——因为 epsilon 细胞在注释里已经存在。scDREAMER 验证它能保留 ionocyte——因为 ionocyte 在注释里已经存在。scIB 用 ARI、NMI、ASW 衡量聚类与注释的一致性——注释是 ground truth。CellANOVA 恢复的是已知条件之间的已知差异。RBET 依赖参考数据集来评估过度校正。

**没有任何方法能检测一个注释里不存在的稀有亚群是否在整合过程中被消灭了。**

如果一个稀有亚群在整合前从未被任何人注释过——它可能藏在某个大群的边缘，只有在足够多的批次汇聚后才有统计能力被识别——那么现有的所有评估框架对它都是盲的。它被消灭了，没有任何指标会下降；它被保留了，也没有任何指标会上升。它在评估体系中不存在。

## 这意味着什么

如果整合操作系统性地消灭未知稀有亚群，那么整个领域基于整合后数据做出的所有下游发现——差异表达分析、细胞通讯推断、轨迹推断、疾病关联——都建立在一个未经检验的假设上：

**整合没有丢掉重要的东西。**

这个假设从未被验证过，因为验证它需要先知道什么被丢了——而如果你知道，它就不是"未知"的了。

## 问题定义

**在单细胞批次整合中，没有任何方法能检测或保护事先未知的稀有亚群。**

这个问题被学界持续关注了至少五年，大量算法和评估框架被提出，但所有工作都局限于已知细胞类型。对于未知稀有亚群，整个领域既没有检测手段，也没有保护机制。

## 解决这个问题所需的学科

机器学习、数学、计算生物学、免疫学、数据科学、肿瘤生物学、生物科学、哲学。

## 什么程度算解决了这个问题

达到以下五个标准，这个问题才算被解决：

**1. 能检测**：存在一个评估框架，能够在不依赖外部标签的情况下，定量检测整合过程是否消灭了某个事先未知的稀有亚群。

**2. 能保护**：存在一个整合算法（或整合策略），在该评估框架下被证明能显著降低未知稀有亚群的丢失率，且不以牺牲主流细胞类型的批次校正效果为代价。

**3. 能扩展**：该框架和算法能处理真实世界规模的数据集。参考标准为 Kang et al.（2024, Nature Communications）构建的泛癌免疫图谱——来自104篇论文、943名患者、30种癌症类型、490万细胞的单细胞数据。这是当前该领域最大规模的整合数据之一，任何声称解决稀有亚群保留问题的方法，如果无法在这个量级上运行，就没有实际意义。

**4. 能验证（计算层面）**：利用现有已发表数据集中已知的稀有细胞类型（如胰腺 epsilon 细胞、肺 ionocyte 等）作为 ground truth，验证该框架确实能检测到这些已知稀有亚群在整合中的丢失。这是现有 benchmark 体系已经具备的验证能力，可以直接复用。

**5. 能验证（生物学层面）**：至少有一个被该框架检测为"在整合中被消灭"的、此前未知的稀有亚群，通过独立的湿实验证实其真实存在并具有生物学功能。这是唯一能从根本上证明该框架不是在检测计算伪影而是在检测真实生物信号的方式。
